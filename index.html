How Mergent Works

Watch how Mergent transforms a raw Pull Request into a perfect merge candidate.

Step 01

Creating the Pull Request

A developer opens a new Pull Request. Mergent picks it up automatically and kicks off the full review pipeline.

Step 02

PR-Issue Linkage

Mergent uses a live vector database and AI-powered semantic similarity to find and link the most relevant GitHub Issue — even if the developer forgot to reference one. It combines text similarity, date proximity, assignee matching, and branch name detection into a multi-signal score.

Step 03

Alignment Validation

After linking a PR to its issue, Mergent analyzes whether the code changes actually implement what the issue asks for. It classifies alignment as Exact, Missing, Tangling, or both — catching scope creep and incomplete implementations before review begins.

Step 04

AI Review Comments

Mergent performs a graph-enhanced code review, catching correctness bugs, security issues, regressions, and broken error handling. Every finding goes through two-pass verification to minimize false positives, and results are posted as inline GitHub review comments with severity levels.

Step 05

PR Enrichment

Every PR receives a comprehensive risk analysis with letter grades (A–E) across nine metrics — from bugs and vulnerabilities to impact size and historically buggy files — plus an interactive dependency graph showing the blast radius of changes.

Step 06

Vibe Reviewing

A Chrome extension that brings an interactive AI review assistant directly onto GitHub PR pages. Chat with AI about the code, draft review comments collaboratively, and verify comment quality — all without leaving the diff view.

End-to-End Workflow

Most agents only post AI comments. Mergent covers every stage — issue linkage, alignment, enrichment, AI review, and vibe reviewing.

  • Full PR lifecycle, not just comments
  • Six pipeline phases working together
  • Configurable per repository

Human in the Loop

Vibe Reviewing is a Chrome extension that puts an interactive AI assistant directly on GitHub PRs — chat, draft comments, and verify quality without leaving the diff.

  • Chat on the diff with exact code context
  • AI drafts become real GitHub comments in one click
  • Persistent multi-thread conversations per PR

Review Quality Check

Before posting, run any review comment through an AI quality gate. It verifies factual accuracy, tone, clarity, and actionability against your repo's custom rubric.

  • Rejects incorrect claims, revises tone, or approves
  • Customizable quality standards per repository
  • Checks directly from the GitHub comment box

Context Engine

A live graph database of every symbol, function, and import chain. Agents get precise context and understand the blast radius of every change.

  • Maps all symbols & dependency chains
  • Knows what a small change can break
  • No wasted tokens on irrelevant code
